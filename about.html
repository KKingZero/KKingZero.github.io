About

I'm Harrison McCall — offensive security engineer, secure systems architect, and founder of Zypheron.

I build automated red-team and AI-driven security tooling. My work spans exploit development, firmware security research, CLI tool design, Active Directory attack chains, and integrating AI into offensive and defensive security workflows.

Zypheron is an AI-native security platform — not another wrapper on top of existing tools. I'm building from the ground up with the assumption that AI agents will be the primary operators of offensive security tooling. That means structured wire protocols, async task queues, machine-consumable APIs, and human approval gates where they actually matter.

What I'm Working On

  • Zypheron Platform — AI-augmented pentesting platform with Active Directory support coming soon
  • Zypheron Desktop — Desktop application for the Zypheron ecosystem (in development)
  • Erebus — AI-native C2 framework built for agent operators, not human terminals
  • Zypheron CLI — Open-source CLI for automated security testing, 30MB binary, no external AI dependencies
  • FirmwareGuard — Open-source firmware integrity and anomaly detection framework

Philosophy

Security should be:

  • Automated — reduce manual overhead, scale testing, eliminate repetitive work
  • Developer-native — tools that fit into real workflows, not toys that demo well
  • AI-augmented, not AI-dependent — AI amplifies the operator, it doesn't replace thinking

The current generation of AI security tools is mostly wrappers. Take existing tools, add an AI layer, call it innovation. That's not what I'm building. I'm rebuilding from scratch — because the architecture of offensive tooling was designed for humans at terminals, and that assumption is about to become obsolete.
